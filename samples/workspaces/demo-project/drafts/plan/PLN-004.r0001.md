## Goal
Make Memora's first ten minutes equal or better than the competing baseline by
letting a user install Memora, attach a local or GitHub repository, import
reviewable evidence, and expose project readiness through MCP without requiring
Docker or manual context assembly.

## Scope
- define import modes and workspace placement
- attach local and GitHub repositories as evidence sources
- import Git history, GitHub issues, pull requests, reviews, and releases as
  reviewable evidence
- apply secret and privacy filtering during import intake
- generate first-run candidate memory and an agent readiness report
- build import progress and baseline approval UI
- expose imported project readiness through MCP
- update the README acquisition story around the legacy-code wedge

## Acceptance Criteria
- M10 issues are executable as a stacked milestone without waiting for merges
- repository import does not write canonical truth without an explicit import
  mode and lifecycle-compliant approval path
- GitHub evidence is treated as evidence, not automatically as interpreted
  project meaning
- users can choose import modes appropriate to small projects or decades-old
  legacy repositories
- MCP can tell agents which project is ready, what evidence exists, and what
  remains ambiguous
- README language sells the legacy-code wedge before installation

## Notes
M10 is intentionally product-heavy because the first-run experience is now a
competitive requirement. If implementation proves too broad, split after
M10-01 and M10-02 establish the import contract instead of weakening the
first-run quality bar.
