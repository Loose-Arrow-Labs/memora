## Constraint
Memora may ingest runtime evidence from external agents and developer tools,
but Memora must not host agent sessions, execute agent workflows, or replay
runtime behavior by re-running tools.

Replay debugging in Memora means reconstructing what happened from captured,
reviewable evidence: prompt inputs, retrieved context, selected artifacts,
tool-call summaries, file diffs, outcomes, and missing-context analysis.

## Why It Exists
Memora's architecture says it is not an execution runtime. Agentmemory's
observability and replay features are compelling, but copying the runtime
boundary would weaken Memora's local-first project-memory model and blur the
line between governed understanding and agent execution.

## Implications
M14 should define a runtime evidence schema and timeline view before adding any
capture integrations. Bug escape analysis should answer what context the agent
had, what it missed, and what artifact should be proposed so the same class of
bug is less likely to escape again.
