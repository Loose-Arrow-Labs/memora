## Constraint
Memora should use app-managed project workspaces as the default storage root.
Source repositories, including local Git folders and GitHub repositories,
should be attached as evidence sources rather than used as the default Memora
workspace root.

Repo-local pointers may be supported later, but canonical Memora artifacts,
drafts, derived indexes, and import receipts should stay isolated from source
code by default.

## Why It Exists
Legacy repositories can be large, messy, sensitive, and long-lived. Keeping
Memora's storage separate reduces accidental source-tree churn, protects
filesystem-first governance, and makes multi-project use easier to reason
about.

## Implications
M10 import should attach a repository to a Memora workspace, scan it as
evidence, and record import state inside Memora's project workspace. Agents
should resolve a Memora project before retrieving context, and each project
should have clearly scoped derived indexes.
