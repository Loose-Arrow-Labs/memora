## Context
Agentmemory showed that the market baseline now includes a strong first-run
experience, visible activity, runtime observability, session replay, and
automatic memory capture. Memora's existing roadmap preserved stronger
governance and deterministic project truth, but those strengths were too hard
to see in the first ten minutes.

The revised strategy keeps Memora's wedge: governed project understanding for
legacy codebases, where agents can respect contribution style, reduce
integration slop, and work from reviewable project evidence.

## Decision
Adopt a revised M10 through M16 roadmap baseline:

1. M10 - Project Import And First Run
2. M11 - Human Review And Trust UI
3. M12 - Existing Repo And GitHub Evidence Understanding MVP
4. M13 - Mobile Contribution MVP
5. M14 - Observability And Replay Debugging
6. M15 - Agentic Workflow Baseline
7. M16 - Remote Conversational Planning

M10 becomes the first-ten-minute product milestone. It should make import from
a local repository or GitHub repository the opening experience, produce
reviewable evidence, and generate a readiness report that tells users what
Memora believes it knows before agents rely on it.

## Alternatives Considered
- keep M10 focused on external workflow integration only
- copy agentmemory's runtime-first product model directly
- defer observability and replay until after remote conversational planning
- make source repositories the default Memora workspace root

## Consequences
The roadmap becomes more ambitious, but better aligned with the product users
will expect after seeing competing memory tools. Memora must now prove value
quickly while preserving its governance boundary: imported evidence can be
bulk-trusted, but extracted meaning remains reviewable unless an explicit
import mode says otherwise.
