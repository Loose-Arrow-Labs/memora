## Context
PBR-16 (#371) reshapes the operator UI to match the filesystem mental model the
user already has from Finder, VS Code Explorer, and Obsidian. The previous
operator shell exposed a flat top-nav of grouped links (Configure, Review, Trust,
Observe) plus a project selector dropdown. A new user opening Memora had no
spatial model of where they were or how a project related to its artifacts.

## Decision
The operator shell adopts a persistent left-pane tree as the primary navigation:

- Tree root is "Memora". Children are each attached project. Each project has
  three section nodes: Agent resources, Artifacts, Project root.
- Each section expands further:
  - Agent resources expands to Context viewer and Understanding output (PBR-19
    will define the full content set).
  - Artifacts expands to Queue, Proposals, All artifacts, Trust dashboard.
  - Project root expands to First-run import status (PBR-20 will define the
    full content set).
- The top bar thins to brand-only. The detailed Review/Observe/Trust/Configure
  groups move into the tree.
- Breadcrumbs appear above the main content area and reflect the current path
  (Memora › Project › Section › Leaf). The most specific item carries
  aria-current="page".
- Tree expansion state persists across sessions via a cookie
  (memora.tree.expanded) updated by a small inline script on details toggle.
- The path to the current selection is always opened automatically by the
  server, regardless of cookie state.
- When zero projects exist, the tree renders the Memora root and an "Add a
  project" CTA that links to /get-started.

## Alternatives Considered
- Keep the flat top-nav and just rename groups to match the new section names.
  Rejected: does not deliver the filesystem mental model the user explicitly
  asked for.
- Use a sidebar drawer or off-canvas pattern instead of a persistent left pane.
  Rejected: hides the navigation, defeats the "always know where you are" goal.
- Implement full keyboard arrow-key navigation in this PR. Deferred: standard
  Tab + Enter on links is the HTML default and keyboard-accessible; arrow-key
  navigation is a follow-up polish.
- Migrate the Context viewer and Understanding routes into the same shell. Out
  of scope for PBR-16; those routes keep their own page layouts in this PR.

## Consequences
- Every operator shell page now renders the tree sidebar; the layout grid is
  flexbox/grid two-column with mobile collapse below 960px.
- HierarchicalSelection and HierarchicalCookieState are public types that each
  Render* method threads through to RenderLayout.
- Two new routes land: /projects/{id}/agent-resources and
  /projects/{id}/project-root. Both are read-only landings honest about PBR-19
  and PBR-20 still defining their content.
- Existing operator-shell test assertions about top-nav group labels ("Review",
  "Observe", "Trust", "Configure") are replaced with tree-pane and breadcrumb
  assertions.
- Context viewer and Understanding routes still render with their own standalone
  page layouts. Unifying them is follow-up scope.
