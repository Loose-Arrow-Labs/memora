## Context
The roadmap and issue text drifted toward saying retrieval was simply
deterministic. That is incomplete. Memora's core context assembly must remain
deterministic and explainable, but the product retrieval strategy is hybrid:
imported evidence, repo scans, GitHub signals, runtime evidence, and later
advisory retrieval can all help discover candidate memory.

The missing distinction made M10 issue #214 and related docs understate the
hybrid retrieval decision.

## Decision
Memora uses hybrid retrieval at the discovery boundary and deterministic
governed assembly at the final context boundary.

Hybrid discovery can find evidence, inferred candidates, and later advisory
candidates. Those candidates must carry provenance, source classification,
confidence, ambiguity, extraction reason, and review state.

Final agent-facing grounded context is assembled from approved or explicitly
allowed Memora artifacts using deterministic, lifecycle-aware rules.

## Alternatives Considered
- describe retrieval as deterministic only
- move semantic or vector retrieval into core v1
- allow advisory discovery to become final context without review
- defer the hybrid distinction until a later retrieval milestone

## Consequences
M10 candidate memory and readiness reporting must distinguish evidence-derived,
inferred, and advisory or future-advisory candidates. Later milestones may add
deeper advisory discovery, but they must not bypass import modes, lifecycle,
approval, provenance, safety filtering, or deterministic grounded assembly.
