## Problem Statement

A developer working with multiple AI agents across multiple tools — ChatGPT,
VS Code, Rider, Obsidian, Codex, Claude — has no shared, governed state that
all of them agree on. Each tool and each agent session starts fresh. Context
is lost between sessions. Agents write without boundaries. The human has to
re-explain, re-orient, and manually intervene when agents go rogue.

## Primary Users / Stakeholders

Solo developers operating a human+agent team across multiple tools and
environments. The operator is the sole approver of canonical truth. Agents
are collaborators, not decision-makers.

## Current Pain

- No shared state between ChatGPT, VS Code/Rider, Obsidian, and agent sessions
- Agents with write access and no planning context go rogue (see OUT-003)
- Approved decisions and constraints must be re-explained every session
- No way to work continuously across environments without losing the thread
- The human has no inbox — no single place to see what agents proposed and
  decide what becomes truth

## Desired Outcome

The operator can move between ChatGPT, VS Code/Rider, Obsidian, and any agent
session and immediately have the same governed project context available in
all of them. Agents propose. The operator approves. Canonical truth is shared
and deterministic. Work is continuous — not session-bounded.

## Definition of Success

- ChatGPT, VS Code/Rider, Obsidian, and all active agents are connected to
  the same Memora workspace
- Proposals from any agent or capture tool appear in a single review inbox,
  accessible from wherever the operator is working
- No agent can write canonical state directly — only propose
- The operator can approve, reject, or escalate from any connected surface
- Reopening any tool or starting any agent session provides immediate project
  context without re-explanation
